### 为什么没生效
`rememberSaveable` 是为**屏幕旋转、进程死亡**设计的。当 Tab 切换导致 `ServerTab` Composable 离开组合树后重新进入，`rememberSaveable` 不保证恢复状态。

logcat 实测：切标签页后 `serverItemsJson` 长度变为 0。

### Claude 的修复（v1.12）
**状态提升**——将 `serverItems`、`serverMessage` 移到 `MainActivity` 层级。Activity 不会因切标签页而销毁，状态自然保留。

### 另外两个 Git 流程问题
1. **没执行 `git merge main`**——你的修改基于 v1.0，当时 main 已经 v1.1 了
2. **没 `git push`**——只 commit 了本地，远程 mimo 分支没更新
